## 📊 配置适用场景

### 当前配置（6000/80）适用于：

```
✅ 100 个渠道均衡负载
✅ 主要渠道（30-40个）QPS < 100
✅ 长尾渠道（60-70个）低频使用
✅ 总 QPS < 5000
```

### 容量分析

```
总连接池: 6000
单渠道最大: 80

理论支持:
- 6000 ÷ 80 = 75 个渠道可同时满载
- 实际使用率: 60%-70%（健康水平）
```

### 极端情况处理

```
如果超过 75 个渠道同时满载:
- MaxConnsPerHost: 0（不限制）
- 会动态创建新连接（超出 80 的部分）
- 新连接用完后会被关闭（不保留在池中）
- 性能稍有下降，但不会报错
```
